Detailed Description

Usage Notes.

A template process.

The order of operation for an algorithm is:

  1. Activate the RunAlgorithm once to initiate the running (usually send first message from initiator)
  2. For each message arrived run ReceiveHandeling

The following are utility methods that can be used for processing:

// Send a message to a destination process
protected void Send(int destProcessId, Message message)
// Send a message to all neighbours (with except list)
protected override void SendToNeighbours(BaseMessage message, List<int> exclude)
// Compose a message and send to all neighbours
protected override void SendToNeighbours(dynamic messageType, AttributeDictionary fields, List<int> exclude = null, int round = 0, int logicalClock = 0)
// Terminate the algorithm (It will cause the whole network to terminate.
// Can be activated from one process or from several processes at the same time
public bool Terminate(Message message = null)

  5. Run - Run the algorithm

  1. For each initiator activate run algorithm
  2. Until termination

So in order to implement an algorithm one should implement:
  1. RunAlgorithm
  2. ReceiveHandeling

Init internal events.

Description.
  • InternalEvent are any events that change the data during the processing
  • The internalEvent has 2 parts:
    1. The Event which holds the condition for activating the InternalEvent
    2. A method that is activated to implement the InternalEvent
  • There are 2 ways to add internal events
    1. Programatically:
      1. Create a method with signature like the DefaultInternalEventHandler
      2. Add insert of the InternalEvent in this method (See the example followed)
    2. Using the GUI
      1. Create a method with signature like the DefaultInternalEventHandler
      2. Add the InternalEvent in the GUI in the design phase
Algorithm.
Usage Notes.
The construct of this method body is:
protected override void InitInternalEvents()
{
// For each event create a call like :
InsertInternalEvent(
Event.EventTrigger.AfterSendMessage, // Trigger
0, // Round
bm.MessageTypes.Forewared, // Message type (can be any message type)
1, // The process which is the other end of the send/receive of the trigger
new AttributeList { "SomeMethod" }); // The name of the method that processes the event
}
