Detailed Description

A message q.

Description.
  • The MessageQ holds a list of messages arrived to a process
  • A message is received by an AsynchronousReader
  • There is an AsynchronousReader for each incoming channel
  • When a message is received by an AsynchronousReader it locks the MessageQ (To avoid concurrent insert of messages) and insert the message to the Q
  • The getting of the messages by the process is done in 2 steps
    1. Retrieve the message
    2. Remove the message from the Q when the process is sure that it wants to process it
  • The MessageQ contains ManualResetEvent that is set when the Q is not empty and reset when the Q is empty. The process processing is done in loop. The loop waits when the Q is empty

◆ ChangeOrder()

public void DistributedAlgorithms.MessageQ.ChangeOrder ( List< int >  permutations,
int  sourceProcessId 
)

Change order.

Description.
This method change the order of messages coming from a process.
Algorithm.
  1. The first step is to create the channelMessagesIdxs which is conversion list from the Q (which contains messages from all processes) to a list that contains messages from the source process
    1. Example:
      1. Suppose we have the following Message Q
        1. [0] Message0 from process 0
        2. [1] Message1 from process 1
        3. [2] Message2 from process 0
        4. [3] Message3 from process 2
      2. The conversion table for process 0 will be
        1. [0] 0
        2. [1] 2
  2. The second step is to create messagesFromProcess which is a list of messages from the source process
    1. For this example the list will be
      1. [0] message0
      2. [1] message2
  3. The third step is to apply the permutations:
    1. Example if we switched between message0 and message1
      1. The permutation list will be
        1. [0] 1
        2. [1] 0
        3. The source index of each change is the value of the entry in permutations
        4. The target index of each change is the index of the entry in permutations
        5. The changes will be:
          1. For index 0 in the permutation list
            1. Take the message in index 1 in the messagesFromProcess list(because 1 is the value)
            2. Put this message in the MessageQ in index channelMessagesIdxs[0] which is 0
          2. For index 1 in the permutation list
            1. Take the message in index 1 in the messagesFromProcess list(because 0 is the value)
            2. Put this message in the MessageQ in index channelMessagesIdxs[1] which is 2.
