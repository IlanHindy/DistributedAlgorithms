Detailed Description

A template process.

Description.
when working there are the following phases
  1. Init - create a new network
  2. Design - Design the network and fill it's parameters
  3. Check - Check if the network design is legal according to the algorithm specifications
  4. Create - Create the network from the software point of view
  5. Run - Run the algorithm
  6. Presentation - update the presentation while running
Usage Notes.
The following is the way the algorithm is run
  1. For each initiator activate run algorithm
  2. Until termination
    1. For each message received activate ReceiveHandeling
So in order to implement an algorithm one should implement:
  1. RunAlgorithm
  2. ReceiveHandeling

Description.
This method performs checks on the channel at the check network phase (before running and after designing)
checking network process.
The check and correct of a network element is composed from :
  1. A CheckBuild method that does the checks and for each error it finds it fills a BuildCorrectionParameters data structure and insert it to a list given as parameter to the method
  2. The BuildCorrectionParameters data structure is composed from :
    1. A message that will be presented
    2. A method that corrects the error (a method with the same prototype as the CorrectionMethod of this class)
    3. An object for passing the parameters from the check method to the correction method.

Init internal events.

Description.
  • InternalEvent are any events that change the data during the processing
  • The internalEvent has 2 parts:
    1. The Event which holds the condition for activating the InternalEvent
    2. A method that is activated to implement the InternalEvent
  • There are 2 ways to add internal events
    1. Programatically:
      1. Create a method with signature like the DefaultInternalEventHandler
      2. Add insert of the InternalEvent in this method (See the example followed)
    2. Using the GUI
      1. Create a method with signature like the DefaultInternalEventHandler
      2. Add the InternalEvent in the GUI in the design phase
Algorithm.
Usage Notes.
The construct of this method body is:
//Example for adding internal events in this method
protected virtual void InitInternalEvents()
{
// For each event create a call like :
InsertInternalEvent(
0, // The id of the process that will activate the event
EventTriggerType.AfterSendMessage, // Trigger (The action the process is doing)
0, // Round (the round of the message)
bm.MessageTypes.Forewared, // Message type (can be any message type)
1, // The process which is the other end of the send/receive of the trigger
new List<InternalEvents.InternalEventDelegate> { SomeMethod }); // The name of the method that processes the event
}

Builds base message.

Description.
  • This method is called in execution time.
  • The purpose of this method is to allow the algorithm to add fields to the Base Algorithm message
  • The following is the process of sending a base algorithm message
    1. During running time before/after a send/receive message the event is checked
    2. If the conditions are filled up this method is called with the base message data
    3. This message can add fields to the Base Message according to the status of the algorithm
    4. The base message is sent
