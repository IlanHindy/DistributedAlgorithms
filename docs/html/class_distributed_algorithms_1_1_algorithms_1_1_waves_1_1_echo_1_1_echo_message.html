Detailed Description

A Echo message.

A template message.

Usage Notes.
The use of this class is in the following way :
  1. Decide what are the types of the messages the algorithm needs
  2. See if you can use the already defined message types in bm.MessageTypes
  3. Fill the enum m.MessageTypes with the messages types
  4. Decide what are the fields needed for all the messages
  5. Fill the m.ork enum with the fields
  6. In order to create a message use one of the constructors
  7. Note that some of the constructors are for the system to be used and you should not use them (because they create a message that does not have the obligatory fields)
  8. Add fields to the message with the method AddField()
  9. Get header fields of the method (fields common to all the messages) using the GetHeaderField() (The header fields enum is bm.pak)
  10. Get private fields of the message using the method GetField() (The private fields enum is m.ork)
Usage Notes.
It is possible to declare other messages classes inherited from the algorithm's main message type (EchoMessage) The type of the message that you handle in the receive has to be the type of the message that was sent (even though the parameter to the process's ReceiveHandeling method gets a EchoMessage as a parameter)

Usage Notes.

There is not actual message type for an algorithm In order to create a message use the DistributedAlgorithms.Algorithms.System.Base.BaseMessage class. The instance of a message is done in the following way:

  1. Decide what are the types of the messages the algorithm needs
  2. Fill the enum MessageTypes with the messages types
  3. Decide what are the fields needed for all the messages
  4. Fill the FieldKeys enum with the fields
  5. In order to create a message use one of the DistributedAlgorithms.Algorithms.System.Base.BaseMessage constructors
  6. Add fields to the message with the method DistributedAlgorithms.Algorithms.System.Base.BaseMessage.AddField()
