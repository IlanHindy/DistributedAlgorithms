Distributed Algorithms
The way the controls presentation in ElementWindow is determined

Introduction

  • The DistributedAlgorithms.ElementWindow is the base window for all the input inserting windows
  • The DistributedAlgorithms.ElementWindow present each attribute in 3 controls
    1. The header TextBox which contains the names (the key) of the field
    2. The existing value TextBox which contains the value currently exist in the NetworkElement
    3. The new value control which contains the new value inserted
  • Each time the window is been build the parameters for the fields presentation has to be determined

The algorithm implementation

Data structures

  1. The class DistributedAlgorithms.ElementWindowPrms is used to hold the presentation parameters
  2. A method of the delegate DistributedAlgorithms.ElementWindowPrmsDelegate is part each attribute make it possible for each attribute to define it's own presentation parameters
  3. The enums DistributedAlgorithms.InputFieldType and DistributedAlgorithms.InputWindows are used to hold the options

The calling chain

  1. The ElementWindow call the attributes's GetElementWindowPrms method
  2. The GetElementWindowPrms method call the method found in the delegate ElementWindowPrmsMethod of the attribute
  3. The ElementWindowPrmsMethod is expected to fill the parameters for each field in the ElementWindow
  4. After that the GetElementWindowPrms check if all the controls properties were set and if not files the data for the control that was not set from a default methods

Definition of the default methods

  1. The attribute handles the default setting for the simple value types
  2. The attribute calls the default setting of the type for the complex types
  3. The definition of the default methods is found in IValueHolder

Changing the behavior

Changing the behavior of all the attributes in a window

  • Change the default method.
  • Do something like this (In the default methods)
    if (inputWindow == InputWindows./* your window */)
    {
    //Set the attributes
    }

Changing the behavior of one attribute

  • Create in the Class that is supposed to hold the attribute a static method from the type of the delegate
  • When adding the attribute set the method to the attribute's ElementWindowPrmsMethod variable
