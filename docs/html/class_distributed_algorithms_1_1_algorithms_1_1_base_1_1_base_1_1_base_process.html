Detailed Description

A base process.

Description.
When working there are the following phases
  1. Init - create a new network
  2. Design - Design the network and fill it's parameters
  3. Check - Check if the network design is leagal according to the algorithm specifications
  4. Create - Create the network from the software point of view
  5. Run - Run the algorithm
  6. Presentation - update the presentation while running

◆ Terminate()

public bool BaseProcess.Terminate ( BaseMessage  message = null)

Terminates the running of the algorithm.

Description.
Algorithm.
The following is the termination process:
  1. Step 1 (TerminationStatus = NotTerminated )
    1. Triggers to move to this step : Received Terminate message, or an activation of the Terminate method
    2. At this stage the following is performed:
      1. Send terminate to all the outgoing channels except for self connecting channel (When the receive thread of the destination will receive the message it will terminate)
  2. Step 2 (TerminationStatus = WaitingForNeighbours)
    1. Trigger to move to this step : The previous step actions were performed
    2. At this step : Receiving Terminate from all the incoming channels except the self connecting channel When this occur the reading thread from the sources terminates
  3. Step 3 (TerminationStatus = WaitingForSelf)
    1. Triggers to move to this step : Revived Terminate from all the incoming channel
    2. At this step Sending Terminate to self and set the StopFlag in order that the listener will terminate
  4. Step 4 (End)
    1. Trigger to move to this step : A message arrived from self
    2. Return true to signal that the process terminated.
Usage Notes.
Note that this method can be activated from one process or several processes the result is the same - termination of all the processes.
