Detailed Description

A network update.

Description.
  • The NetworkUpdate algorithm is used in the following case:
    1. A network was designed
    2. The automatically generated code of the algorithm was changed
    3. When reentering the program we want to update the network (To merge between the code changes and the existing changes)
  • The algorithm
    1. For each component in the existing network
      1. Create a new component (with the new code)
      2. Deep scan of the element and:
        1. If the was changed when building the network - copy the value to the new network
        2. If the attribute is a list or dictionary do a clever merge (see the methods for the algorithm of the merge

Gets existing child attribute.

Description.
There can be the following cases
  1. The attribute is found in both lists/dictionaries (So the attribute will be found)
  2. The attribute is found only in the exists list In this case There was a merge between the lists that puts the attributes of the existing in the new so the attribute will be found

Merge lists.

Description.
  • Clever merge between the existing and new lists
    • If the attribute is found in the existing and the new
      • Copy the value from the existing to the new only if the value of the attribute was changed during network build
    • If the attribute is found in the existing and not the new
      • Add the attribute tree to the new
    • If (Default case - automatically happens) The attribute exists in the new and not the old
      • keep it in the new
  • The attributes are identified using the idsInList
