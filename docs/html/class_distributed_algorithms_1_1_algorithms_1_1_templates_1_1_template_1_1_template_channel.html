Detailed Description

A template channel.

Description.
when working there are the following phases
  1. Init - create a new network
  2. Design - Design the network and fill it's parameters
  3. Check - Check if the network design is legal according to the algorithm specifications
  4. Create - Create the network from the software point of view
  5. Run - Run the algorithm
  6. Presentation - update the presentation while running
Usage Notes.
  1. The channel represents a directed channel between 2 processors
  2. The channel can be accessed from the source process's OutChannels() list
  3. The channel can be accessed from the destination processor's InChannels() list
  4. The channel's or dictionary can hold data specific to the channel (which the processes put in)
  5. After each running phase the channel is drawn on the screen. In order to change the presentation change the attributes of the pp dictionary using the bc.ppk keys
  6. Note that removing attributes from the pp dictionary is dangerous because the presentation of the channel Use them.

Check build.

Description.
This method performs checks on the channel at the check network phase (before running and after designing)
checking network process.
The check and correct of a network element is composed from :
  1. A CheckBuild method that does the checks and for each error it finds it fills a BuildCorrectionParameters data structure and insert it to a list given as parameter to the method
  2. The BuildCorrectionParameters data structure is composed from :
    1. A message that will be presented
    2. A method that corrects the error (a method with the same prototype as the CorrectionMethod of this class)
    3. An object for passing the parameters from the check method to the correction method.
Algorithm.

Correct an error detected during check build.

Description.
  • This method is an example for the methods that needs to be created for correcting build errors.
  • For each error there has to be a method from this prototype that corrects the error.
Algorithm.
Usage Notes.
  1. Create a correction method for each error with a method with this prototype
  2. In the CheckBuild when encountering an error set the correction method name in the BuildCorrectionParameters data structures
  3. The methods gets its parameters from the object parameter. The structure of the parameters has to be the same in the CheckBuild method and the correction method
  4. In order to access element of the network, Start with the private network member of this class.
  5. The processes of the network are in network.Processes
  6. The channels of the network are in network.Channels
