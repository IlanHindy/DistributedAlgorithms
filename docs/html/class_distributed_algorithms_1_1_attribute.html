Detailed Description

An attribute.

Description.
This class is the only value holder class in the application.
Usage Notes.
The attribute has 5 public members
  1. The value (dynamic)
  2. Editable : In ElementWindow whether the value of the attribute can be changed
  3. IncludedInShortDescription : In several usage whether to include the attribute
  4. EndInputOperation : In ElementWindow actions (like checking) for activating when the value is changed
  5. ElementWindowPrmsMethod : For ElementWindow - the parameters that sets the way the controls in the ElementWindow will be created.

Determine if the new value can be assigned to the type of the value that already exist in the attribute.

Description.
A new value can be assigned if :
  1. The value of the attribute is null
  2. The existing value is primitive and one of the following is true
    1. The value of the attribute is number and so the value to be assigned
    2. The new value can be assigned to the existing value (This is done by taking the string of the new value and try to parse it using the type of the existing value)
  3. If the value is a string and can be parsed to value from the type of the current value If the type of the value and the current value is the same.

Gets ElementWindow prms.

Description.
  1. The ElementWindow is the base window for presenting/changing attributes of NetworkElements
  2. The ElementWindow is composed from 3 controls for each attribute
    1. Header TextBox which usually contains the key
    2. Existing value TextBox which contains the existing values
    3. New Value control which contains controls to change the value
  3. Each attribute has a ElementWindowPrmsMethod in which the input values for the controls can be assigned
  4. In the method the user can set the prms for the ElementWindow controls
  5. In the method the user can set each value he wants
  6. This method does the followings:
    1. Calls the delegate method
    2. Fills default values to all the values that the delegate method did not fill.
