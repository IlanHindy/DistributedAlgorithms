Detailed Description

A template channel.

Description.

when working there are the following phases

  1. Init - create a new networ
  2. Design - Designe the network and fill it's parameters
  3. Check - Check if the network design is leagal according to the algorithm specifications
  4. Create - Create the network fro the software point of view
  5. Run - Run the algorithm
  6. Presentation - update the presentation while running

Usage Notes.

  1. The channel represents a directed channel between 2 processors
  2. The channel can be accessed from the source processir's OutGoingChannels list
  3. The channel can be accessed from the destination processor's InCommingChannels list
  4. The channel's dictionaries can hold data specific to the channel (which the processes put in)
  5. After each running phase the channel is drawn on the screen in order to change the presentation change the attributes of the PresentationParameters using the PresentationParametersKeys
  6. Note that there is no effect to add attributes from the PresentationParameter dictionary
  7. Note that removing attributes from the PresentationParameters is dangerouse because thepresentation Use them.

phase (before running and after designing)

checking network process. The check and correct of a network element is composed

from :

  1. A CheckBuild method that does the checks and for each error it finds it fills a BuildCorrectionParameters data structure and insert it to a list given as parameter to the method
  2. The BuildCorrectionParameters data structure is composed from :
  3. A message that will be presented
  4. A method that correctes the error
  5. An object for passing the parameters from the check method to the correction method.
