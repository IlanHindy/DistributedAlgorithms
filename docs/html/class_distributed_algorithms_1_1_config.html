◆ LoadConfig()

public void DistributedAlgorithms.Config.LoadConfig ( )

Loads the configuration.

Description.
  • This method is used to Load the config when the program starts
  • It does :
    1. Load the config from the config file
    2. If the config file do not exist it asks for a config file to load
    3. if the load of the config file failed it fills the missing data
    4. The missing data is divided to 2 parts:
      1. The main (PrivateAttribute) dictionary which is filled from the backup (ElementAttributes)
      2. The Algorithm Data directory reading
      3. If the reading of the Algorithm Data directory failed the method generates a new one
Algorithm.

(static Dictionary<dynamic,RoutedEventHandler>) - The buttons click events. The Following is the mechanism of connecting buttons event handler to the buttons

  • The event handlers are found in the ConfigWindow (because they are activated by the dialog
  • The attributes that use buttons have a ElementWindowPrmsMethod ButtonControlSetting
  • So when the button is defined (means the dialog tree is generated the ButtonControlSetting is called
  • The ButtonControlSetting connects the button event handler to the button according to the buttonsClickEvents dictionary.
  • So when the button is pressed the event handler from the ConfigWindow is activated
